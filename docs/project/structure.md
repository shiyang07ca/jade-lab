# 目录结构

仓库顶层目录表达内容生命周期，内层目录表达该类内容最稳定的定位方式。

| 目录 | 内层组织轴 | 准入条件 |
| --- | --- | --- |
| `cookbook/` | 语言 → 主题 | 已理解、可重复验证、准备长期复用 |
| `problems/` | 平台 → 题目 → 语言 | 具有稳定平台和题目标识符 |
| `labs/` | 领域 → 独立项目 | 有明确研究目标和独立运行环境 |
| `tools/` | 工具名称 | 有稳定输入输出、错误语义和验证 |
| `templates/` | 语言 → 工程类型 | 能重复生成可运行工程；按需创建 |
| `studies/` | 项目名称 | 外部开源源码，必须由 submodule 管理 |
| `docs/` | 研究主题 | 结论、约定、设计决策和路线图 |

目录名使用小写 kebab-case；语言包名遵循对应生态约定。禁止新建 `misc`、`others`、`playground`、`temp` 等模糊目录。空分类目录不进入版本库。
