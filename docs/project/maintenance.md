# 日常维护工作流

## 开始工作

先选择一个独立模块，阅读其 README，恢复该模块自己的环境，并写下本次要验证的问题和预期结果。一次工作不跨越多个无关模块。

## 内容晋升

临时验证先放在本地 `.scratch/`。形成明确研究目标后进入 `labs/`；得到稳定、可复用的实现后进入 `cookbook/` 或 `tools/`；能够重复创建同类工程时再形成 `templates/`。没有继续维护价值的内容直接删除。

## OJ

LeetCode 题目由 leetgo 维护，按题目标识符聚合语言。竞赛题先保存在 `contests/`，获得正式题号后迁入标准题目目录并删除副本。

## 周期整理

每周检查新 lab、未完成标记、空目录和无用依赖，决定继续、晋升或删除。每月按模块升级依赖并清理虚拟环境、缓存和构建产物；`references/` 中 submodule 的版本更新独立评估。
