# 维护约定

本仓库采用单主题、小步迁移的维护方式。

新增内容前先判断其归属：稳定语言样本进入 `cookbook/`，OJ 进入 `problems/`，探索性项目进入 `labs/`，可重复使用的程序进入 `tools/`，外部参考源码进入 `references/`。

每次维护应限定在一个可独立验证的模块内。依赖声明与锁文件同步更新；目录迁移、行为修改和依赖升级尽量分开。实验没有继续维护价值时直接删除，由 Git 历史承担归档职责。

不得提交凭证、虚拟环境、构建产物、编辑器状态或个人临时文件。submodule 中不记录个人笔记，研究结论写入 `docs/open-source/`。
